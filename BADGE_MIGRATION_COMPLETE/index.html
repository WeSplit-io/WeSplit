    üîÑ How It Works Now¶

    Badge Loading Flow¶

    1. App starts ‚Üí badgeService.getUserBadgeProgress()
    2. Loads from Firestore ‚Üí loadFirestoreBadges() (cached 5 min)
    3. Merges with config ‚Üí Firestore badges override config
    4. Returns progress ‚Üí Includes badgeInfo from database

    Badge Claim Flow¶

    1. User enters code ‚Üí claimEventBadge(redeemCode)
    2. Looks up badge ‚Üí getBadgeInfoByRedeemCode() (Firestore first)
    3. Claims badge ‚Üí Saves to users/{userId}/badges/{badgeId}
    4. Updates user ‚Üí Adds to users.badges[] array

    Badge Display Flow¶

    1. Component needs badge info ‚Üí Calls badgeService.getBadgeInfoPublic()
    2. Or uses progress ‚Üí BadgeProgress.badgeInfo (already loaded)
    3. Displays badge ‚Üí Uses data from database
