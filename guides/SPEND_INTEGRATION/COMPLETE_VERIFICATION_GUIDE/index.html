    SPEND Integration - Complete Verification Guide¶

    Date: 2025-01-27
    Purpose: Comprehensive guide to verify split creation, email invitations, deep links, and fallback behaviors

    üéØ Overview¶

    This guide helps you verify that the complete SPEND integration flow works correctly:

    1. ‚úÖ Split Creation: Splits are created and stored in Firestore
    2. ‚úÖ Email Invitations: Emails are sent with proper deep links
    3. ‚úÖ Deep Links: Both app-scheme and universal links work correctly
    4. ‚úÖ Fallback Behaviors: Web pages work when app is not installed
    5. ‚úÖ End-to-End Flow: Complete flow from creation to payment

    3. Deep Link Generation ‚úÖ¶

    What to Verify: - [ ] Universal links use correct domain (wesplit-deeplinks.web.app) - [ ] App-scheme links use correct format (wesplit://) - [ ] URLs are properly encoded - [ ] Deep links work for both view-split and spend-callback

    How to Verify:

    Spend Callback Deep Links: 

    # Universal link
https://wesplit-deeplinks.web.app/spend-callback?callbackUrl=ENCODED_URL&orderId=ORDER_ID&status=success

# App-scheme link
wesplit://spend-callback?callbackUrl=ENCODED_URL&orderId=ORDER_ID&status=success

    Invite Links: 

    # Should use: wesplit-deeplinks.web.app
https://wesplit-deeplinks.web.app/join-split?invite=BASE64_ENCODED_DATA

    Expected Result: 

    ‚úÖ Universal Link Domain: Uses correct domain
‚úÖ App-Scheme Link: Format correct
‚úÖ Spend Callback Deep Link: Both formats generated
‚úÖ URL Encoding: URLs properly encoded

    üìß Email Invitation Flow¶

    Complete Flow¶

    1. SPEND calls batchInviteParticipants
   ‚Üì
2. WeSplit processes invitations:
   - Existing users ‚Üí Added directly to split
   - New users ‚Üí Pending invitation created
   ‚Üì
3. For new users:
   - Generate invite link: https://wesplit-deeplinks.web.app/join-split?invite=...
   - Send email with invite link
   - Store pending invitation in Firestore
   ‚Üì
4. User receives email
   ‚Üì
5. User clicks link:
   - If app installed ‚Üí Opens in app (universal link)
   - If app not installed ‚Üí Shows web page ‚Üí Redirects to app store
   ‚Üì
6. User creates account and joins split

    Invite Link Format¶

    Generated Link: 

    https://wesplit-deeplinks.web.app/join-split?invite={base64_encoded_data}

    Encoded Data Contains: 

    {
  "email": "user@example.com",
  "splitId": "split_123",
  "timestamp": 1234567890
}

    üß™ Testing Scenarios¶

    Scenario 1: New User Invitation¶

    Steps: 1. Create split via API 2. Invite new user (email not in WeSplit) 3. Check email is sent 4. Click invite link 5. Verify web page loads 6. Verify app opens (if installed) 7. Verify user can join split

    Expected: - ‚úÖ Email sent - ‚úÖ Invite link uses wesplit-deeplinks.web.app - ‚úÖ Web page accessible - ‚úÖ App opens from link - ‚úÖ User can join split

    Scenario 2: Existing User Invitation¶

    Steps: 1. Create split via API 2. Invite existing user (email in WeSplit) 3. Check user is added directly 4. Verify no email sent (or notification sent)

    Expected: - ‚úÖ User added directly to split - ‚úÖ No pending invitation created - ‚úÖ Push notification sent (if implemented)

    Scenario 3: App Not Installed¶

    Steps: 1. Open invite link on device without app 2. Verify web page loads 3. Verify app store links shown 4. Install app 5. Open link again 6. Verify app opens

    Expected: - ‚úÖ Web page loads - ‚úÖ App store links visible - ‚úÖ After install, app opens from link

    Scenario 4: Deep Link Callback¶

    Steps: 1. Complete payment in WeSplit 2. Verify callback deep link generated 3. Open callback link 4. Verify redirects to SPEND app

    Expected: - ‚úÖ Callback deep link generated - ‚úÖ Link uses correct domain - ‚úÖ Redirects to SPEND app

    Common Issues¶

    Issue: Deep links not in response - Cause: Split doesn't have callbackUrl in externalMetadata - Fix: Ensure callbackUrl is provided when creating split

    Issue: Emails not sent - Cause: Email service not configured - Fix: Set EMAIL_USER and EMAIL_PASSWORD Firebase Secrets

    Issue: Web pages return 301 - Cause: Firebase hosting redirects - Fix: Check firebase.json rewrite rules

    Issue: Invite links use wrong domain - Cause: Old code using wesplit.io - Fix: Ensure generateInviteLinkSync uses wesplit-deeplinks.web.app

    üö® Troubleshooting¶

    Emails Not Sending¶

    1. Check Email Service: 

      firebase functions:secrets:access EMAIL_USER
firebase functions:secrets:access EMAIL_PASSWORD

    2. Check Logs: 

      firebase functions:log --only batchInviteParticipants

    3. Verify Transporter:

    4. Check Gmail credentials
    5. Verify "Less secure app access" or App Password

    Deep Links Not Working¶

    1. Check Domain:
    2. Verify wesplit-deeplinks.web.app is correct

    3. Check firebase.json configuration

    4. Check Universal Links:

    5. Verify apple-app-site-association file

    6. Verify assetlinks.json file

    7. Check App Configuration:

    8. Verify app.config.js has correct domains
    9. Rebuild app after changes

    Web Pages Not Loading¶

    1. Check Firebase Hosting: 

      firebase hosting:channel:list

    2. Check Rewrite Rules:

    3. Verify firebase.json has correct rewrites
    4. Deploy hosting: firebase deploy --only hosting:deeplinks
