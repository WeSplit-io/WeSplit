    Complete Split Flow Audit: Creation ‚Üí Funding ‚Üí Withdrawal¶

    üîç Flow Overview¶

    1. CREATION
   ‚îú‚îÄ‚îÄ Split Document Created (splits collection)
   ‚îú‚îÄ‚îÄ Split Wallet Created (splitWallets collection)
   ‚îú‚îÄ‚îÄ Private Key Stored (SecureStore/Firebase)
   ‚îî‚îÄ‚îÄ Split Document Updated with walletId/walletAddress

2. FUNDING
   ‚îú‚îÄ‚îÄ Fair Split: processParticipantPayment()
   ‚îÇ   ‚îú‚îÄ‚îÄ Check user balance
   ‚îÇ   ‚îú‚îÄ‚îÄ Execute transaction (fair_split_contribution)
   ‚îÇ   ‚îú‚îÄ‚îÄ Update participant status to 'paid'
   ‚îÇ   ‚îî‚îÄ‚îÄ Sync to splits collection
   ‚îÇ
   ‚îî‚îÄ‚îÄ Degen Split: processDegenFundLocking()
       ‚îú‚îÄ‚îÄ Check user balance
       ‚îú‚îÄ‚îÄ Execute transaction (degen_split_lock)
       ‚îú‚îÄ‚îÄ Update participant status to 'locked'
       ‚îî‚îÄ‚îÄ Sync to splits collection

3. WITHDRAWAL
   ‚îú‚îÄ‚îÄ Fair Split: extractFairSplitFunds()
   ‚îÇ   ‚îú‚îÄ‚îÄ Verify creator
   ‚îÇ   ‚îú‚îÄ‚îÄ Verify balance
   ‚îÇ   ‚îú‚îÄ‚îÄ Execute withdrawal (UnifiedWithdrawalService)
   ‚îÇ   ‚îú‚îÄ‚îÄ Update wallet status to 'completed'
   ‚îÇ   ‚îî‚îÄ‚îÄ Save transaction
   ‚îÇ
   ‚îú‚îÄ‚îÄ Degen Winner: processDegenWinnerPayout()
   ‚îÇ   ‚îú‚îÄ‚îÄ Verify winner
   ‚îÇ   ‚îú‚îÄ‚îÄ Verify balance
   ‚îÇ   ‚îú‚îÄ‚îÄ Execute withdrawal (CentralizedTransactionHandler)
   ‚îÇ   ‚îú‚îÄ‚îÄ Update participant to 'paid'
   ‚îÇ   ‚îú‚îÄ‚îÄ Update wallet status
   ‚îÇ   ‚îî‚îÄ‚îÄ Save transaction
   ‚îÇ
   ‚îî‚îÄ‚îÄ Degen Loser: processDegenLoserPayment()
       ‚îú‚îÄ‚îÄ Verify loser
       ‚îú‚îÄ‚îÄ Verify balance
       ‚îú‚îÄ‚îÄ Get external destination
       ‚îú‚îÄ‚îÄ Execute withdrawal (CentralizedTransactionHandler)
       ‚îú‚îÄ‚îÄ Update participant to 'paid'
       ‚îú‚îÄ‚îÄ Update wallet status
       ‚îî‚îÄ‚îÄ Save transaction

    ‚ùå Issues Found¶

    1. Static Imports in SplitWalletPayments.ts¶

    Location: Lines 9-10, 12 

    import { doc, updateDoc, collection, getDoc, getDocs, query, where } from 'firebase/firestore';
import { db } from '../../config/firebase/firebase';
import { BalanceUtils } from '../shared/balanceUtils';

    Problem: These static imports cause Metro bundler to analyze and bundle Firebase SDK during initial bundling, contributing to memory crashes.

    Impact: Memory crashes during bundling, especially when handlers are loaded.

    Fix: Convert to dynamic imports.

    7. Inconsistent Transaction Execution¶

    Problem: Different handlers use different services for withdrawals: - FairSplitWithdrawalHandler uses UnifiedWithdrawalService.withdraw() - DegenWinnerPayoutHandler uses CentralizedTransactionHandler.executeTransaction() - DegenLoserPaymentHandler uses CentralizedTransactionHandler.executeTransaction() - TransferHandlers.sendToCastAccount() uses CentralizedTransactionHandler.executeTransaction() - TransferHandlers.transferToUserWallet() uses UnifiedWithdrawalService.withdraw()

    Impact: Inconsistent behavior, harder to maintain, different error handling.

    Fix: Standardize on UnifiedWithdrawalService.withdraw() for all withdrawals.

    ‚úÖ Recommended Fixes¶

    Priority 1: Fix Static Imports¶

    2. Convert BalanceUtils import to dynamic

    Priority 2: Create Shared Helper Functions¶

    1. getAndValidateWallet() - Wallet retrieval + validation
    2. findParticipant() - Participant finding + validation
    3. updateParticipantInList() - Participant update logic
    4. validateWalletBalance() - Balance verification
    5. saveWithdrawalTransaction() - Transaction saving
    6. updateWalletStatusAndSync() - Wallet status update + sync

    Priority 3: Standardize Transaction Execution¶

    1. Use UnifiedWithdrawalService.withdraw() for all withdrawals
    2. Remove inconsistent CentralizedTransactionHandler.executeTransaction() calls for withdrawals

    Priority 4: Consolidate Common Patterns¶

    1. Extract common validation logic
    2. Extract common error handling
    3. Create shared types for common parameters
